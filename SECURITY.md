# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**提交公开 Issue。

请发送邮件至项目维护者：**zhengmark@users.noreply.github.com**

我们会：

- 在 48 小时内确认收到报告
- 在 7 天内提供初步评估和修复计划
- 在修复发布前保持沟通
- 在修复发布后公开披露（如适用）

## 安全实践

本项目遵循以下安全实践：

- **密钥管理**：所有 API 密钥通过环境变量注入，不硬编码在代码中
- **依赖安全**：Dependabot 自动监控依赖漏洞并提 PR 更新
- **密码存储**：用户密码使用 bcrypt 哈希存储
- **JWT 安全**：使用强密钥签名，设置合理过期时间
- **环境隔离**：`.env` 文件已加入 `.gitignore`，不提交到仓库
- **输入验证**：所有用户输入经过 Pydantic 模型验证
