## 二、方向清单（按投入产出比排序）

### 🟢 低投入高产出（本周可上）

#### 1. 前端展示约束违规警告
- **现状**：后端 `constraint_checker.py` 已检测违规并追加到 narration 末尾（`⚠️ 注意`），前端不需要改就能看到。但格式不显眼。
- **做法**：前端解析 result 中 `violations` 字段（通过 `/api/plan` 返回），展示为醒目的黄色警告卡片
- **工作量**：2h（server.py 返回 violations + index.html 渲染）
- **效果**：用户一眼看到"这条路线有3个问题"

#### 2. 用户画像面板
- **现状**：`UserProfileManager` 已积累 interests/favorites/history，但用户完全看不到
- **做法**：加一个 `/api/profile` GET/PUT + 前端"我的画像"侧栏，展示兴趣标签、收藏地点、出行历史
- **工作量**：3h
- **效果**：用户感知"系统认识我"，增强粘性

#### 3. 规划进度真实百分比
- **现状**：SSE 推了 progress 事件（`📍城市` / `🧠Intent` / `🔧工具`），但前端只列文字不显示进度条
- **做法**：定义 8 步固定阶段，每阶段推百分比，前端渲染进度条
- **工作量**：1h
- **效果**：等待焦虑降低，专业感提升

#### 4. 路线分享链接
- **现状**：路线规划完只能自己看
- **做法**：生成 `/share/{id}` 短链接，存 JSON 到 `data/shares/`，展示静态路线页（无交互）
- **工作量**：3h
- **效果**：社交传播入口

---

### 🟡 中投入中产出（下周）

#### 5. 多方案对比（2-3 条路线）
- **现状**：每次只返回一条路线
- **做法**：`route_agent.py` 在 `query_clusters` 返回后让 LLM 选 2 组不同的 cluster 组合，各调一次 `build_route`，返回多条路线供用户选择
- **工作量**：5h
- **关键风险**：增加 LLM token 消耗 + 耗时
- **效果**：用户有选择权，满意度 +20%

#### 6. 移动端响应式适配
- **现状**：PC 端可用，手机端布局错乱（Leaflet + Mermaid 不响应）
- **做法**：
  - CSS media query 适配（地图/文字上下堆叠而非左右并排）
  - 输入区 sticky 底部（移动端键盘弹出时保持可见）
  - Leaflet 手势优化（移动端双指缩放）
- **工作量**：8h
- **效果**：覆盖手机用户场景（出行时用手机查路线）

#### 7. 会话持久化（Redis / 文件）
- **现状**：`sessions` 是内存 dict，服务重启全丢
- **做法**：最简单 — session 序列化到 `data/sessions/{id}.json`，启动时恢复
- **工作量**：2h
- **效果**：多轮对话跨重启不丢失

#### 8. 用户反馈闭环
- **现状**：路线产出后无反馈渠道
- **做法**：每轮路线下方加 👍/👎 按钮，结果写入 `data/feedback/`，定期分析差评路线的共性
- **工作量**：3h
- **效果**：积累数据驱动优化，形成飞轮

---

### 🔴 高投入高产出（里程碑）

#### 9. 营业时间实时校验
- **问题**：用户说"晚上11点宵夜"，系统推荐了晚上9点关门的店
- **做法**：`tool_build_route` 选出 POI 后，调高德 `place/detail` API 查营业时间，过滤掉不营业的
- **工作量**：5h + API 配额消耗
- **效果**：深夜场景准确率大幅提升

#### 10. 天气感知推荐
- **问题**：下雨天推荐户外公园
- **做法**：接入和风天气 API（免费），在 system prompt 注入天气上下文 → LLM 自动调整推荐
- **工作量**：4h（含 API 注册）
- **效果**：用户觉得"AI 真贴心"

#### 11. 单元测试基础设施
- **现状**：0 单元测试，全靠集成测试烧 API 配额
- **做法**：
  - `graph_planner` 单测（haversine 距离 + 最短路径正确性）
  - `constraint_checker` 单测（34 条规则全覆盖）
  - `cluster_tools` 单测（黑名单/关键词映射/去重缓存）
  - Mock LLM 响应 + Mock Amap 响应
- **工作量**：12h
- **效果**：改动不再靠烧配额验证，开发速度 +50%

#### 12. Docker 部署 + 环境标准化
- **现状**：裸 Python 进程，依赖系统级 pip
- **做法**：Dockerfile + docker-compose（FastAPI + SQLite volume）
- **工作量**：3h
- **效果**：一键部署到任何服务器

## 四、不做/暂缓

|------|------|
