# Claude Code 模式指南（小白版）

## 什么是"模式"？

Claude Code 有不同的工作模式，就像手机有"正常模式"和"省电模式"一样。不同模式适合不同场景。

---

## 1. 默认模式

**干嘛的：** 你问什么，我就做什么。每个操作（比如修改文件、执行命令）都会弹窗让你确认。

**怎么开：** 不用开，默认就是这个。

**适合：** 日常写代码，不着急，每步都确认。

---

## 2. Auto Mode（自动模式）

**干嘛的：** 不用每次都弹窗问你了，我自己决定能不能做。省去频繁点"允许"的烦恼。

**怎么开：** 你需要先配置好权限（告诉系统哪些操作可以直接做），然后输入 `/auto`。

**适合：** 你信任我，想做快一点。

> ⚠️ 别担心，危险操作（删文件、push 代码）还是会问你的。

---

## 3. Plan Mode（计划模式）

**干嘛的：** 我先探索代码、设计方案，**不写任何代码**。方案给你看，你点头了我再动手。

**怎么开：** 输入 `/plan`，或者告诉我一个大需求，我会主动进入。

**适合：** 大改动、新功能、不确定怎么实现的时候。先规划再动手，不返工。

---

## 4. Fast Mode（快速模式）

**干嘛的：** 用更快的模型，输出速度更快，但思考深度不变。

**怎么开：** 输入 `/fast`，再输一次关掉。

**适合：** 简单任务，想要快一点。

---

## 5. Caveman Mode（极简模式）

**干嘛的：** 我说话更短，去掉废话，只留关键信息。省 token，速度快。

**怎么开：** 输入 `/caveman`。

**适合：** 你嫌我话太多的时候。或者你在手机上用，屏幕小。

---

## 6. Worktree Mode（隔离模式）

**干嘛的：** 新建一个临时分支干活，不影响你主分支的代码。等做好了再合并。

**怎么开：** 跟我说"用 worktree 做 xxx"就行。

**适合：** 实验性改动，不确定能不能行，想先试试看。

---

## 7. Loop Mode（循环模式）

**干嘛的：** 每隔一段时间自动执行一个任务，比如"每 5 分钟检查一下部署好了没"。

**怎么开：** 输入 `/loop 5m 检查部署状态`（5m = 5分钟，10m = 10分钟）。

**适合：** 等 CI 构建、等部署完成、定期检查状态。

---

## 快速对照表

| 我想... | 用这个 |
|---------|--------|
| 正常干活 | 默认模式 |
| 减少弹窗确认 | Auto Mode `/auto` |
| 先看方案再动手 | Plan Mode `/plan` |
| 快点输出 | Fast Mode `/fast` |
| 别废话 | Caveman Mode `/caveman` |
| 不影响主分支 | Worktree |
| 定时执行任务 | Loop Mode `/loop` |
