## 7 项修复详情

### P0-1：解说/数据脱节
- **文件**：`route_agent.py` system prompt
- **改动**：新增"解说数据一致性"规则 — 强制每个 POI 名称必须是 `build_route` 返回 `stops[].name` 的精确值；不匹配时如实告知而非编造
- **预期效果**：U1/U3 的解说不会再说"蝎王府羊蝎子"但 stops 实为 KTV

### P0-2：品类错配
- **文件**：`cluster_tools.py`
- **改动**：
  - 新增 `_KEYWORD_CATEGORY_MAP` — 28 个关键词→品类映射
  - 新增 `_cluster_relevance()` — 相关性评分 0.0-1.0
  - `query_clusters` 返回增加 `keyword_match` 字段，按相关性排序，筛掉 `match=0` 的簇
- **预期效果**：U1 搜索"美食"时不会返回 KTV/艺术空间簇

### P0-3：极简输入放弃
- **文件**：`route_agent.py` system prompt
- **改动**：新增"极简输入处理规则"段落 — 无起点→geocode 城市中心，无关键词→`["美食","景点"]`，无预算→medium
- **预期效果**："西安 吃"直接规划，不再反问

### P1-1：build_route 盲目重试
- **文件**：`cluster_tools.py`
- **改动**：`_DEDUP_CACHE` — 30 秒 TTL 去重缓存，相同 cluster_ids+num_stops 返回缓存结果
- **预期效果**：U3 对同一 cluster_ids 调 5 次 → 只执行 1 次

### P1-2：多轮坐标系丢失
- **文件**：`route_agent.py` `run_unified_agent` + `_build_context`
- **改动**：
  - session 中的 origin_coords/dest_coords 注入 agent_state
  - `_build_context` 告知 LLM "坐标已缓存，地点未变可跳过 geocode"
- **预期效果**：U1 三轮不再每轮重复 geocode("钟楼")

### P2：API 限流无退避
- **文件**：`llm_client.py`
- **改动**：`_retry_request()` 3 次指数退避（2s→4s→8s）
- **预期效果**：偶发 429 时自动恢复

### Bugfix：_retry_request 的 falsy 陷阱
- **文件**：`llm_client.py`
- **根因**：`requests.Response.__bool__()` 对 4xx/5xx 返回 `False`，导致 `if last_error:` 判断失败，函数返回 None
- **修复**：改为 `if last_error is not None:`
- **教训**：与项目 0.0 坐标 bug 同类 —— Python falsy 检查必须用 `is not None`
