# POI 搜索 + 可视化升级方案

## 目标

1. **POI 搜索引擎**：基于倒排索引的毫秒级 POI 全文搜索
2. **交互式备选**：搜索结果可作为备选 POI 加入/移出路线
3. **可视化升级**：路线展示更美观，地图更具信息量

## 架构改造

```
用户输入 → 路线规划（不变）
         → 新增：搜索框 → 倒排索引 → POI 结果 → 地图气泡 + 侧边栏
                    ↓
              可拖拽加入路线 / 点击移除
```

## 第一步：倒排索引 (后端)

### 数据结构

```python
# 内存中的倒排索引
class InvertedIndex:
    tokens: dict[str, list[int]]      # 词 → poi_id 列表
    poi_by_id: dict[int, dict]        # poi_id → {name, lat, lng, category, rating, ...}
    
    def build():    # 启动时从 DB 加载全量 15k POI
    def search(q):  # AND 查询，返回匹配的 poi 列表
```

### 分词策略

- POI 名称：按字切 + 2-gram（如"大雁塔" → ["大","雁","塔","大雁","雁塔"]）
- 品类：按 `;` 拆分后每段按字切
- 拼音无关，纯中文分词

### 查询算法

```
输入 "回民街小吃"
  → 分词: ["回民","民街","小吃"]
  → 对每个词取 inverted lists
  → 求交集 → poi_ids
  → 按 rating DESC + name length ASC 排序
  → 返回 top 20
```

### 性能

- 15k POI × 平均名长 5 字 → ~50k token entries → < 5MB 内存
- 查询 O(k × log n) → ~0.1ms

### 交互流程

1. 用户在搜索框输入关键词 → 200ms debounce → 调 API
2. 搜索结果展示在侧边栏"备选区"，同时在地图上打半透明标记
3. 点击 [+ 加入路线] → 该 POI 加到当前路线，自动重排
4. 点击地图上标记 → 弹出气泡：名称 + 评分 + [加入]/[移除]
5. 加入后立即更新路线折线和侧边栏站点列表

### 视觉增强

- 站点标记按顺序标号（① ② ③ ... ⑥ 终点）
- 不同类型用不同颜色：起点绿、美食橙、景点蓝、终点红
- 搜索结果用半透明灰色标记区分
- 路线折线用渐变色（起点→终点 绿→红）
- 侧边栏卡片式布局，带 POI 缩略信息

## 第四步：备选 POI 加入路线逻辑

### 后端新增 `POST /api/route/add-poi`

```json
{
  "session_id": "abc123",
  "poi_id": 1234
}
```

→ 后端把该 POI 插入当前路线的最优位置（按 projection 排序）
→ 返回更新后的路线数据

### 前端本地处理

更轻量的方案：前端维护本地 `alternativePois` 列表，直接更新地图显示。确认路线时把 `added_pois` 一起提交。

## 实施顺序

1. **倒排索引** → 构建 + 测试搜索速度
2. **搜索 API** → 端点 + 集成到 server
3. **前端搜索框** → UI + debounce + API 调用
4. **地图标记** → 搜索结果可视化
5. **备选加入** → 交互逻辑
6. **视觉美化** → 彩色标记、布局调整

## 风险

- 倒排索引内存 ~5MB，可接受
- 搜索 debounce 200ms 避免刷 API
- 加入 POI 后路线需重新投影排序（用现有 projection 逻辑）

## 验收

- 输入"回民街" → 0.5s 内展示搜索结果
- 搜索结果显示在地图上
- 点击加入 → POI 出现在路线中
- 界面美观，信息层次清晰
