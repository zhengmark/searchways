## 角色与职责

### 首席维护者

- 审查和合并 Pull Request
- 管理发布周期
- 处理安全漏洞报告
- 维护项目路线图
- 管理社区行为准则

## 成为维护者

我们欢迎活跃的贡献者成为维护者。条件：

1. 持续贡献至少 3 个月
2. 合并过至少 3 个 PR
3. 熟悉项目架构和代码规范
4. 现有维护者一致同意

## 维护者承诺

- 在 48 小时内回复 Issue 和 PR
- 在决策前征求社区意见
- 遵循项目行为准则
